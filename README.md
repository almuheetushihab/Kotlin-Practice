# Kotlin MVVM Practice Project 🚀

এটি একটি প্রফেশনাল অ্যান্ড্রয়েড প্রজেক্ট যা **Clean Architecture**, **MVVM Pattern**, এবং আধুনিক অ্যান্ড্রয়েড টেকনোলজি ব্যবহার করে তৈরি করা হয়েছে। এই অ্যাপটির মূল উদ্দেশ্য হলো—কীভাবে ডেটা একাধিক সোর্স (Remote API, Room Database, DataStore) থেকে হ্যান্ডেল করতে হয় এবং সেগুলোকে একটি রিঅ্যাক্টিভ ইউজার ইন্টারফেসে (Jetpack Compose) দেখাতে হয়।

---

## 🛠 টেকনোলজি স্ট্যাক (Tech Stack)

এই প্রজেক্টে আধুনিক অ্যান্ড্রয়েড ডেভেলপমেন্টের সেরা টুলসগুলো ব্যবহার করা হয়েছে:

- **Kotlin:** অ্যাপের মূল প্রোগ্রামিং ল্যাঙ্গুয়েজ।
- **Jetpack Compose:** আধুনিক ডিক্লারেটিভ UI কিট।
- **MVVM Architecture:** ডেটা এবং UI-কে আলাদা রাখার জন্য ব্যবহৃত হয়েছে।
- **Clean Architecture:** কোডকে মেনটেইনেবল এবং স্কেলেবল করার জন্য লেয়ারড আর্কিটেকচার।
- **Dagger Hilt:** ডিপেন্ডেন্সি ইনজেকশনের জন্য।
- **Retrofit & OkHttp:** নেটওয়ার্কিং বা API কলের জন্য।
- **Room Database:** লোকাল ডেটা পারসিস্টেন্স বা ক্যাশিংয়ের জন্য।
- **DataStore (Preferences):** ছোট ছোট কী-ভ্যালু ডেটা (যেমন ইউজার সেটিংস) সেভ করার জন্য।
- **Kotlin Coroutines & Flow:** অ্যাসিঙ্ক্রোনাস প্রোগ্রামিং এবং ডেটা স্ট্রিমের জন্য।

---

## 🏗 আর্কিটেকচার লেয়ার (Architecture Layers)

অ্যাপটি তিনটি প্রধান লেয়ারে বিভক্ত:

1.  **Data Layer:** 
    - `ApiService`: সার্ভার থেকে ডেটা আনে।
    - `UserDao`: Room ডাটাবেজে ডেটা সেভ করে।
    - `UserPreferences`: DataStore-এ ডেটা সেভ করে।
    - `UserRepository`: API, Room এবং DataStore-এর মধ্যে সমন্বয়কারী হিসেবে কাজ করে।

2.  **Domain Layer (Optional Logic):**
    - এখানে ডেটা মডেল (`User`) এবং বিজনেস লজিক থাকে।

3.  **UI Layer:**
    - `UserViewModel`: UI-এর জন্য প্রয়োজনীয় স্টেট এবং লজিক হ্যান্ডেল করে।
    - `UserScreen`: Jetpack Compose ব্যবহার করে বানানো ইউজার ইন্টারফেস।

---

## ⚙️ অ্যাপটি কীভাবে কাজ করে? (Workflow)

১. **ডেটা সোর্সিং:** অ্যাপটি একই সাথে **Room Database** এবং **DataStore** থেকে ডেটা রিড করে এবং UI-তে দেখায়।
২. **রিফ্রেশ (API Call):** যখন ইউজার "Refresh" বাটনে ক্লিক করে, তখন `UserViewModel` একটি API কল ট্রিগার করে।
৩. **ডেটা সিঙ্কিং:** সার্ভার থেকে ডেটা আসার পর, সেটি সরাসরি Room Database এবং DataStore-এ সেভ করা হয়।
৪. **রিঅ্যাক্টিভ আপডেট:** যেহেতু আমরা `Flow` এবং `StateFlow` ব্যবহার করেছি, তাই ডাটাবেজে ডেটা সেভ হওয়া মাত্রই UI অটোমেটিক আপডেট হয়ে যায়।
৫. **এরর হ্যান্ডেলিং:** যদি ইন্টারনেট না থাকে বা API কল ফেইল করে, তবে অ্যাপটি ক্রাশ করবে না। পরিবর্তে এটি একটি `UIState.Error` থ্রো করবে এবং ইউজারকে Snackbar-এর মাধ্যমে মেসেজ দেখাবে।

## 🚦 UI স্টেট ম্যানেজমেন্ট

আমরা একটি `Sealed Class` ব্যবহার করে UI স্টেট কন্ট্রোল করি:
- **Idle:** প্রাথমিক অবস্থা।
- **Loading:** ডেটা যখন সার্ভার থেকে লোড হচ্ছে (ProgressBar দেখায়)।
- **Success:** সফলভাবে ডেটা আপডেট হলে।
- **Error:** কোনো সমস্যা হলে ইউজারকে মেসেজ দেখায়।
